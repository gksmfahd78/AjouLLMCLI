# AjouLLM CLI

Node.js CLI/TUI for using Mindlogic Gateway models from the terminal. It supports one-shot chat, fullscreen TUI sessions, project context scanning, planning, and agent-style file edits.

## Highlights

- Terminal chat with streaming and non-streaming responses
- Fullscreen TUI for interactive sessions
- Project context indexing for code-aware prompts
- Planning mode for file-aware implementation plans
- Agent mode for model-generated full-file edits
- Local config through environment variables or `.ajoullmrc.json`

## Screenshots

Screenshot and GIF slots are prepared in [`docs/screenshots`](docs/screenshots). Add terminal captures there when recording the CLI/TUI flow.

## Requirements

- Node.js 18.17 or newer
- Mindlogic Gateway API key

## Installation

```powershell
npm.cmd install
```

To use `ajoullm` as a global command:

```powershell
npm.cmd link
ajoullm
```

Without a global link on Windows, run the CLI through:

```powershell
.\ajoullm.cmd
```

## Quick Start

```powershell
.\ajoullm.cmd config init
$env:AJOULLM_API_KEY="YOUR_API_KEY"
.\ajoullm.cmd config set model gpt-5.4-nano
.\ajoullm.cmd
```

You can also store the API key in the project config:

```powershell
.\ajoullm.cmd config set apiKey YOUR_API_KEY
```

For shared machines or long-lived environments, prefer `AJOULLM_API_KEY` over storing secrets in `.ajoullmrc.json`.

## Common Commands

```powershell
.\ajoullm.cmd
.\ajoullm.cmd "hello"
.\ajoullm.cmd chat "hello"
Get-Content .\notes.txt | .\ajoullm.cmd
node .\bin\ajoullm.js "hello"
```

Chat options:

```powershell
.\ajoullm.cmd chat --model gpt-5-mini "summarize this"
.\ajoullm.cmd chat --system "Answer briefly in Korean." "hello"
.\ajoullm.cmd chat --stream "streaming response"
.\ajoullm.cmd chat --no-stream "non-streaming response"
.\ajoullm.cmd chat --temperature 0.3 --top-p 1 --max-tokens 4096 "question"
```

## TUI

```powershell
.\ajoullm.cmd
.\ajoullm.cmd interactive
```

Useful keys:

- `Enter`: submit current input
- `Ctrl+N` or `Shift+Enter`: add a new line
- `Up` / `Down`: navigate input history
- `PageUp` / `PageDown`: scroll logs
- `Tab`: slash commands or `@file` autocomplete
- `Esc`: clear input or stop an active run
- `Ctrl+C`: exit

Slash commands:

```text
/help
/mode chat|context|plan|edit|agent
/apikey <key>
/model <name>
/system <text>
/stream on|off
/init
/compact
/undo
/export [file]
/credits
/models
/status
/clear
/exit
```

## Project Context

```powershell
.\ajoullm.cmd code scan
.\ajoullm.cmd code context "fix login redirect bug"
.\ajoullm.cmd code plan "fix login redirect bug"
.\ajoullm.cmd code cache show
.\ajoullm.cmd code cache clear
```

- `code scan`: indexes project files and writes `.ajoullm/project-context.json`
- `code context`: prints a concise context bundle for a task
- `code plan`: generates a file-aware implementation plan when an API key is available
- `code cache show`: prints the current context cache JSON
- `code cache clear`: removes the cached project context

## Agent Mode

```powershell
.\ajoullm.cmd agent --plan "fix login redirect bug"
.\ajoullm.cmd agent --apply "fix login redirect bug"
.\ajoullm.cmd agent --apply --dry-run "refactor config loading"
.\ajoullm.cmd agent --apply --verify "npm.cmd test" "fix failing tests"
.\ajoullm.cmd agent --resume
```

Notes:

- `--plan` stores an implementation plan in `.ajoullm/agent-session.json`
- `--apply` applies model-generated full-file edits
- `--dry-run` lists intended file changes without writing them
- `--verify <command>` runs a verification command after edits
- `--resume` prints the latest agent session summary

## Configuration

```powershell
.\ajoullm.cmd config init
.\ajoullm.cmd config show
.\ajoullm.cmd config path
.\ajoullm.cmd config set apiKey YOUR_API_KEY
.\ajoullm.cmd config set model gpt-5.4-nano
.\ajoullm.cmd config set baseUrl https://factchat-cloud.mindlogic.ai/v1/gateway
.\ajoullm.cmd config set systemPrompt "Answer in Korean."
.\ajoullm.cmd config set temperature 0.3
.\ajoullm.cmd config set topP 1
.\ajoullm.cmd config set maxTokens 2048
.\ajoullm.cmd config set stream false
.\ajoullm.cmd config set reviewModel gpt-5.4-nano
.\ajoullm.cmd config set reviewFallback false
.\ajoullm.cmd config unset systemPrompt
.\ajoullm.cmd config reset
```

Supported environment variables:

- `AJOULLM_API_KEY`
- `AJOULLM_MODEL`
- `AJOULLM_BASE_URL`
- `AJOULLM_SYSTEM_PROMPT`
- `AJOULLM_TEMPERATURE`
- `AJOULLM_TOP_P`
- `AJOULLM_MAX_TOKENS`
- `AJOULLM_STREAM`
- `AJOULLM_REVIEW_MODEL`
- `AJOULLM_REVIEW_FALLBACK`
- `AJOULLM_REQUEST_TIMEOUT_MS`

## Generated Local Files

- `.ajoullmrc.json`: local project config
- `.ajoullm/project-context.json`: project context index
- `.ajoullm/agent-session.json`: latest agent session
- `.ajoullm/tui-session.json`: latest TUI session/input history
- `ajoullm.md`: project note generated by `/init`
